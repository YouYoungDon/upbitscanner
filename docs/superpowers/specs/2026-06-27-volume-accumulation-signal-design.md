# 거래량 선행 매집 시그널 설계 (2026-06-27)

## 배경
기존 `거래량 급증` 신호(lib/signals.mjs)는 **전일比 +2% 상승 후에만** 매수 점수를 부여(L116) →
엘프(06-27)처럼 거래량이 폭발해도 가격이 이미 +33% 오른 뒤에야 고득점 → **후행**.
오르기 전 "저점 매집" 단계를 잡기 위한 신규 선행 신호를 추가한다.

## 신규 시그널: `거래량 선행 매집`
세 조건 동시 충족 시 발동:

| 조건 | 기준 |
|------|------|
| 과매도 | Stoch K < 20 **또는** RSI < 30 |
| 가격 횡보 | 전일比 변화율 ∈ (-3%, +2%) |
| 거래량 미세 증가 | volR ≥ 1.5x (= 당일 거래량 ÷ 20일 평균, `calcVolRatio`) |

- 발동 시 라벨 `거래량 선행 매집 (1.8x)`, **기본점수 2** (addBuy).
- 기존 `거래량 급증(매수)`는 +2% 상승 요구 → 횡보(<+2%) 구간만 잡는 이 신호와 **상호 배타적**.
- 지금까지 놓치던 "volR≥2인데 가격 미상승이라 점수 0"이던 매집 초기를 포착.

## 노이즈 통제
과매도 게이트가 핵심. 거래량 1.5x는 흔하나 "과매도 + 횡보"까지 동시 충족은 드물어 헛신호가 적음.

## 튜닝
`SIGNAL_KEYS`에 `'거래량 선행 매집'` 추가 → weekly.mjs가 적중률 자동 추적·가중치 조정.
초기 보수적 2점, 데이터 축적 후 weekly가 조정.

## 범위 밖(이번엔 안 함)
- 콤보 보너스 연동(`거래량 급증` 전용 유지), 떨어지는칼 필터 연동 — 추후 데이터 보고 결정.

## 테스트
lib/signals 단위테스트에 케이스 추가:
1. 과매도+횡보+volR 1.5x↑ → `거래량 선행 매집` 발동
2. +2% 이상 상승 → 발동 안 함(기존 `거래량 급증`이 담당)
3. 과매도 아님 → 발동 안 함
4. 거래량 1.5x 미만 → 발동 안 함
